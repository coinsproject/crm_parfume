Инструкции по обновлению системы

← К документации

Автоматизированный процесс обновления системы на сервере с проверками и резервным копированием.

Автоматическое обновление

Для обновления системы используйте скрипт full_update.sh:

cd /opt/crm_parfume
sudo ./full_update.sh
Что делает скрипт:
  1. Резервное копирование базы данных - создает копию перед обновлением
  2. Обновление кода из Git - получает последние изменения из репозитория
  3. Анализ изменений - определяет необходимость пересборки Docker образа
  4. Пересборка образа (если нужно) - собирает новый Docker образ с обновленным кодом
  5. Перезапуск контейнера - останавливает и запускает контейнер с новым образом
  6. Применение миграций - автоматически применяет все миграции базы данных
  7. Проверка прав доступа - проверяет и исправляет права в базе данных
  8. Синхронизация ролей - синхронизирует описания ролей и права
  9. Проверка работоспособности - проверяет, что приложение работает корректно
Примечание: Скрипт автоматически обрабатывает локальные изменения в Git и конфликты при обновлении.

Резервное копирование

Скрипт автоматически создает резервную копию базы данных перед обновлением. Копии хранятся в директории ./backups/.

Автоматически удаляются старые копии, оставляются только последние 10.

Ручное обновление (если автоматическое не работает)

Шаг 1: Резервное копирование
cd /opt/crm_parfume
sudo docker cp parfume-crm:/app/data/crm.db ./backups/crm_backup_$(date +%Y%m%d_%H%M%S).db
Шаг 2: Обновление кода
sudo git pull origin main
Шаг 3: Пересборка образа
sudo docker-compose build
Шаг 4: Перезапуск контейнера
sudo docker-compose down
sudo docker-compose up -d
Шаг 5: Применение миграций
sudo docker-compose exec crm alembic upgrade head
Шаг 6: Проверка работоспособности
curl http://localhost:8000/health

Решение проблем

Проблема: Конфликт локальных изменений

Если при обновлении возникает конфликт локальных изменений:

sudo git reset --hard HEAD
sudo git pull origin main
Проблема: Миграции не применяются

Проверьте текущую версию миграций:

sudo docker-compose exec crm alembic current

Примените миграции вручную:

sudo docker-compose exec crm alembic upgrade head
Проблема: Приложение не запускается

Проверьте логи контейнера:

sudo docker logs parfume-crm --tail=100

Проверьте статус контейнера:

sudo docker ps -a
